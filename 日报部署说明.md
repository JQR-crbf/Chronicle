### 默认路径（自动创建）

```

Linux:   ~/Documents/Chronicle/日报/

在应用中点击"设置保存路径"按钮，选择任意目录。

## 🎯 **使用方法**

### 1️⃣ **生成日报**

1. 打开应用，进入 **Insights** 视图
2. 选择日期，查看当天活动
3. 点击 **"生成日报"** 按钮
4. 查看并编辑生成的日报内容

### 2️⃣ **推送到 GitHub**

1. 点击 **"推送日报到 GitHub"** 按钮
2. 在弹出窗口中输入：
   - **GitHub Personal Access Token**
   - **成员名称**（如：张三）
   - **团队目录**（如：开发团队 dev-team）
3. 点击 **"确认推送"**
4. 等待推送完成 ✅

日报会自动推送到：
```
成员日志 members/{团队目录}/{成员名称}/{日期}_log.md
```

### 3️⃣ **自定义保存路径（可选）**

1. 在应用设置中找到 **"日报保存设置"**
2. 点击 **"选择新的保存目录"**
3. 选择你想要的文件夹
4. 点击确认 ✅

---

## 🔑 **获取 GitHub Personal Access Token**

### 步骤：

1. 访问：https://github.com/settings/tokens
2. 点击 **"Generate new token (classic)"**
3. 设置名称（如：Chronicle App）
4. 勾选权限：**`repo`**（必需）
5. 点击 **"Generate token"**
6. **复制 Token**（只显示一次！）
7. 在应用中使用

### 注意事项：

- ✅ Token 不会被应用保存，每次推送需要输入
- ✅ 确保有 `repo` 权限才能推送文件
- ✅ Token 泄露后立即在 GitHub 撤销并重新生成

## ❓ **常见问题**

### **Q: 需要安装 Python 吗？**
A: **不需要！** v2.0 已完全移除 Python 依赖 ✅

### **Q: 需要安装依赖包吗？**
A: **不需要！** 应用内置所有功能 ✅

### **Q: 如何更改保存路径？**
A: 在应用设置中点击"日报保存设置"，选择新路径 ✅

### **Q: 推送失败，提示认证错误**
A: 检查 GitHub PAT 是否正确，是否有 `repo` 权限

### **Q: 如何查看推送的日报？**
A: 访问 GitHub 仓库：`AIEC-Team/AIEC-agent-hub`，查看对应路径

### **Q: 可以推送到其他 GitHub 仓库吗？**
A: 目前固定推送到 `AIEC-Team/AIEC-agent-hub`，如需修改请联系开发者

### **Q: 本地日报文件在哪里？**
A: 默认在 `~/Documents/Chronicle/日报/`，也可以自定义

## 📦 **打包分发**

### 分发文件：

```
